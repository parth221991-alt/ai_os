## Important Constraints

1. **Live users**: Any regression affects paying subscribers. Fix regressions before new features.
2. **Razorpay live keys**: Must not be exposed in git history or client-side code.
3. **Zerodha rate limits**: kite-auto-login and market-snapshot must respect Kite Connect API limits.
4. **Supabase project**: Schema changes require Supabase migrations, not ad-hoc SQL.
5. **Groq dependency**: Currently a tech debt — new AI features must use Anthropic, not Groq.

---

## Architectural Decisions

- **Supabase as backend**: Eliminates custom auth and DB management for a SaaS product. Correct choice for a solo founder.
- **Edge Functions for AI calls**: Keeps API keys server-side. The Groq key should have been here from the start — it ended up in frontend by mistake.
- **Groq instead of Anthropic**: Historical decision — Groq was faster/cheaper for early prototyping. Migration to Claude is backlog item (medium priority). New features must use Claude.
- **pg_cron for automation**: Avoids separate cron server. Depends on Supabase's pg_cron support — confirm extension is enabled.
- **React 19 + Radix UI**: Aligned with canonical AI_OS frontend stack.

---

## Known Risks

1. **Groq API key in frontend**: Most critical security issue. Exposed to every browser that loads the app.
2. **Groq dependency**: If Groq API goes down, all AI features fail. No fallback.
3. **Single Supabase project**: No staging environment documented. Schema changes go directly to production.
4. **pg_cron rate limits**: market-snapshot runs every minute — must verify Kite Connect API allows this without hitting quotas.
5. **TOTP automation for Zerodha**: kite-auto-login uses TOTP — if the TOTP mechanism changes (2FA policy update), auto-login breaks silently.

## Important Learnings

- **Security first on SaaS**: API keys in frontend `.env` with `REACT_APP_` prefix are sent to the browser. This is a fundamental React/CRA misconception — all sensitive keys must be in server-side functions.
- **Supabase Edge Functions are the right abstraction**: For a solo founder, they eliminate the need for a custom API server for AI calls, auth middleware, and scheduled jobs.
- **Tech debt compounds**: The Groq decision created a security risk (key in frontend) and a migration burden. Future AI integrations must go through Claude from day one.
