# F10 — COMMENT REACT

**Formula ID:** F10  
**Type:** Community-Driven Q&A / Comment Mining  
**Reel Duration:** 20–30 seconds  
**Production Complexity:** Low — no tool demo required, script derived from an existing comment  
**Repeat Frequency:** High — use 2× per week once comments are available (requires 500+ follower base to have comment volume)  
**Primary CTA:** Comment engagement + Follow  
**Derived from:** APIFY Week 4 "ENGAGEMENT" theme + competitor intelligence showing comment-mining content drives 3-5× comment rate vs standard content (comment begets comment — the algorithm rewards the loop)

---

## Purpose

Turns a real viewer comment or question into a reel that responds directly to it. The formula does two things simultaneously: it answers the specific question for the commenter (and everyone else who had the same question but didn't ask), and it signals to the broader audience that @ai_snipp is a responsive, listening creator — not a broadcast channel.

**Why it works:** When a viewer sees their comment featured in a reel, they share it. When other viewers see that the creator responds to comments with full reels, they comment more. The algorithm detects the increased comment rate and boosts the next post. This is a flywheel that compounds over time.

**Psychological mechanism:** "My comment was seen" (for the commenter) + "This creator responds to people like me" (for the audience) + increased likelihood to comment → algorithm boost → new audience.

**Prerequisites:** This formula requires real comments to mine. Use it after there is sufficient comment volume on existing posts. In the early growth phase (under 500 followers), substitute with a "frequently asked in DMs" framing — same format, slightly different attribution.

---

## Best Use Cases

- Questions about a tool shown in a previous reel ("how do I get access to this?")
- Confusion about a concept from a previous video ("lekin ye AI replace karega na hum logon ko?")
- Requests for deeper content ("bhai ye ek nahi, 10 tools bata do")
- Debate/disagreement in comments ("ChatGPT ye nahi karta tha — tum galat ho")
- Questions that reveal a common gap ("main naya hoon, kahan se shuru karun?")
- Positive reactions that can be amplified ("yaar ye actually kaam aaya!")
- Questions that deserve a full tutorial (the question is simple, the answer is a mini-tutorial)

**Avoid for:** Comments that are off-topic or would confuse the audience if featured. Negative/trolling comments — do not amplify negativity. Comments where the answer is genuinely too complex for a 30-second reel. Comments that name other creators negatively.

## Hook Structures

**Hook Type A — The Direct Call-Out Hook (most powerful)**
```
"[Username] ne comment kiya tha — 
'[The comment, verbatim]'

Ye acha sawaal tha. 
Seedha jawaab deta hoon."
```

**Hook Type B — The "Many of You" Hook**
```
"Bahut logon ne poocha — 
'[The common question]'

Main [topic] pe ek full reel bana raha tha,
lekin pehle ye seedha answer karta hoon."
```

**Hook Type C — The Disagreement Hook**
```
"[Username] ko mujhse agree nahi karna —
aur unka point valid hai.

Lekin ye bhi sach hai — [counter-point].
Dono sides seedha bol raha hoon."
```

**Hook Type D — The DM Feature Hook**
```
"DM mein kisi ne poocha —
'[The question from DM]'

Itna acha sawaal tha ki reel banana pada.
Suno."
```

**Hook Type E — The "Let's Test This" Hook**
```
"[Username] ne kaha — '[Skeptical claim or question]'

Theek hai. Main abhi live test karta hoon.
Dekho kya hota hai."
```

**Selection rule:** Type A for single standout comments that deserve a full response. Type B when multiple comments ask the same thing (common question = audience gap). Type C for productive disagreements — shows the creator can hold nuance. Type D for DM questions (early stage or when comment volume is low). Type E for skeptical or challenge comments — live proof is the best response.

## Story Structure

```
[0-4s]   HOOK        → Feature the comment. Name the commenter (or "bahut logon ne"). Create anticipation.
[4-8s]   VALIDATE    → Acknowledge why the question is good or the concern is valid. One sentence.
[8-22s]  ANSWER      → The actual answer, demonstration, or rebuttal. This is the reel's main content.
[22-26s] SYNTHESIS   → One line that closes the answer. The takeaway. Not a summary — the bottom line.
[26-30s] CTA         → "Tumhara sawaal hai? Comment karo — main jawaab dunga."
```

**The VALIDATE beat prevents creator arrogance.** Jumping directly from "you asked this" to "here's the answer" feels like a lecture. One sentence acknowledging the question or concern ("Ye actually ek solid question hai") creates warmth before the answer. It also signals: the creator has read and understood the comment, not just used it as a content vehicle.

## Scene Structure

| Clip | Block | Duration | Content |
|---|---|---|---|
| Clip 0 (Comment Card) | No creator — comment visual | 3s | The actual comment displayed as a card/screenshot |
| Clip 1 | Block 1 — HOOK (conversational) | 5s | Creator references the comment, validates, sets up answer |
| Clip 2 | Block 6 — STORYTELLING (Answer) | 10s | The core answer — creator eye level, conversational |
| Clip 3 | Block 3 or 2 — PROOF/DEMO | 8s | Optional — demo or example supporting the answer |
| Clip 4 | Block 5 — CTA | 5s | Comment CTA + follow |

**Comment card treatment (Clip 0):** The comment appears as a clean card — dark background, the comment text in white, the username in a lighter color, an Instagram comment bubble aesthetic. This is added in post (Gemini Omni or Canva). It signals immediately: "this is a response to a real comment."

**For the "many of you" variant (Type B Hook):** Skip Clip 0 (no specific comment to show). Start directly with Creator on full screen.

---

## Character Usage

**Clip 0:** Creator not present. The comment card IS the opening content.

**Clip 1:** CONVERSATIONAL direction — the most informal of all formulas. This is a direct reply to a specific person. The character should feel like they're responding in a DM or a face-to-face conversation. Slightly casual body language (may be leaning forward slightly, more relaxed than standard). The validate line should feel genuine: "Ye actually solid sawaal hai" — not performative.

**Clip 2 (Answer):** Eye level. The most natural, unscripted-feeling delivery in the formula library. The character is talking to the commenter specifically — which creates the intimacy that makes other viewers feel like they're overhearing a helpful conversation. Hand gestures are welcome here — this is the least gesture-restricted formula (no finger counting still, but natural conversation-appropriate gestures are fine).

**Clip 3 (Demo if applicable):** Facecam overlay with the relevant tool or example on screen. The character occasionally glances at the screen as if checking their own demonstration — authentic, not scripted.

**Clip 4 (CTA):** The comment invitation is the primary CTA — the character delivers it with genuine openness. "Aur tumhara sawaal hai? Neeche comment karo — main seriously padhta hoon aur jawaab dunga." This should feel like a real promise, not a standard CTA.

## Camera Language

| Clip | Camera | Movement | Notes |
|---|---|---|---|
| Comment Card | Full frame — the comment text | Static | Readable, clean. No animation needed. |
| Hook | Eye level (conversational — not authority) | Gentle push-in (2% over 5s) | More casual than standard — peer, not authority |
| Answer | Eye level | Static or very slow drift | Informal, like a conversation |
| Demo (optional) | Facecam 25% bottom-right | Static on demo | Demo must be legible |
| CTA | Eye level | None | Genuine comment invitation delivery |

**Note on camera position:** This is the ONLY formula in the AI_SNIPP library where the hook camera is at true eye level, not slightly below. The slightly-below-eye framing signals authority. This formula signals peer-to-peer conversation — equal, not expert-to-novice. This is an intentional choice.

## Editing Style

- **Comment card design:** Instagram comment aesthetic — rounded rectangle with subtle shadow, dark background, username in a lighter color, comment text in white. Optional: include the like count on the comment (if impressive). Should look like a real comment screenshot, not a designed graphic.
- **Username highlight:** When creator mentions the commenter's name in Clip 1, the comment card briefly re-appears as a bottom-third for 1.5 seconds. Keeps the attribution clear.
- **Answer visual support:** If the answer includes a specific claim or stat, add a brief text card (white on dark, 2 seconds) showing the key information. This keeps dense answers visually scannable.
- **No timer elements, no side-by-side:** This formula is conversation-first. All the visual devices from other formulas (timers, split screens, comparison cards) are NOT used here. Clean, simple, creator-forward.
- **Color grade:** The warmest grade in the formula library. This is community content — the visual treatment should feel approachable and warm.
- **Transition into answer:** A single clean cut from the Hook clip to the Answer clip. No transition effects — the reply should feel immediate, not produced.

---

## Audio Style

- **Music:** Lower volume than all other formulas (40% instead of standard 60-70%). This formula is a conversation — the music is atmospheric, not structural. The creator voice is primary.
- **Music choice:** Warm, ambient, low energy. Not the upbeat electronic of F04/F07. Not cinematic of F03. Think: background music in a coffee shop conversation.
- **No SFX:** The Comment React formula uses NO special SFX. No impact hits, no riser, no keyboard sounds. The absence of production SFX signals authenticity — this is a reply, not a production.
- **Comment notification sound (optional):** At the very start (when the comment card appears), a single soft notification ping — subtle, mobile notification energy. 0.5s. Then silence until music begins under Clip 1. This one exception to the no-SFX rule is acceptable.
- **Creator voice:** Slowest, most natural pace of all formulas. The conversational delivery should feel unhurried. The viewer is listening to a reply, not consuming a tutorial.

**Primary — Comment Invitation:**
```
"Aur tumhara sawaal hai? 
Comment karo neeche — main genuinely padhta hoon.
Acha sawaal ho toh reel banta hai isi pe."
```

**Secondary — Follow for More Answers:**
```
"Aise aur Q&A ke liye follow karo @ai_snipp —
tumhara sawaal agle reel mein ho sakta hai."
```

**Tertiary — DM for Private Questions:**
```
"Agar sawaal public karna nahi chahte —

```

**The primary CTA creates the flywheel.** "Acha sawaal ho toh reel banta hai isi pe" — this sentence is key. It tells the audience that commenting is not just engagement, it's a content brief submission. This upgrades the comment relationship from reaction to collaboration.

## Flow Generation Strategy

**This formula requires the least Flow investment of any formula.** The primary asset is creator performance — the comment mining and the conversational reply. Tool UI is optional. The production investment should be in script writing, not clip generation.

**Comment selection criteria before generating any clip:**
1. The question is one that many others would benefit from hearing answered
2. The answer is deliverable in 25 seconds or less
3. The answer demonstrates AI knowledge (stays on brand)
4. The commenter seems genuine and well-intentioned
5. Featuring this comment will encourage similar comments (not trolling behavior)

**Comment card (Clip 0):** NOT a Flow clip — designed in Canva or Gemini Omni as a static image. Template should be ready in the editing toolkit so this can be produced quickly. The faster you respond to a comment, the higher the original commenter's engagement.

**Clip 1:** Block 6 — STORYTELLING (not Block 1 HOOK). The conversational delivery of this formula does not fit the "authority hook" framing. Key addition to prompt: "This is a direct reply to a viewer comment — the character's body language is open, relaxed, and genuinely engaged. Not a performance. A conversation. The character may have a slight 'good question' expression — warm acknowledgment before the answer."

**Clip 2 (Answer):** Block 6. This is the longest creator-speaking clip in this formula. The delivery should feel natural — allow occasional mid-sentence pauses where the character is "finding the words" (this authenticity reads as unscripted, which is valuable here). Add: "Natural, slightly informal delivery — this is a DM reply elevated to a reel. Not scripted-feeling."

**Clip 3 (Demo — optional):** Block 3 if needed, Block 2 if showing a reference. Only include if the answer genuinely benefits from demonstration. Do not force a tool demo for the sake of production quality.

**Clip 4 (CTA):** Block 5. Comment invitation delivery. Add: "The character delivers the comment invitation with genuine openness — slightly forward lean, warm expression. This is an invitation to a conversation, not a call-to-action performance."
