# F05 — CAREER LIST

**Formula ID:** F05  
**Type:** Curated List — Career / Education Value  
**Reel Duration:** 28–35 seconds (slightly longer — depth earns runtime)  
**Production Complexity:** Medium — multiple facecam beats with screen demos  
**Repeat Frequency:** Medium — use 1× per week, always with fresh resources  
**Primary CTA:** Follow (@ai_snipp) + optional DM trigger  
**Derived from:** AI_SNIPP_REEL.md (proven formula — the AI Certifications reel is the canonical example of this formula working)

---

## Purpose

Delivers a curated list of 3–5 AI resources, certifications, tools, or skills that have genuine career impact. Unlike the Hidden Tool formula which showcases one thing, this formula builds the creator's reputation as a curator — someone who has done the research and distilled it to the most valuable items.

**Why it works:** The Indian tech audience has a strong relationship with structured learning paths. Lists feel actionable. Certifications and career resources generate saves — viewers treat this content as reference material, which drives the algorithm.

**Psychological mechanism:** "I don't know where to start" anxiety → structured list → immediate clarity → save for later → follow for more lists.

**Unique advantage:** This formula is proven to drive saves and shares more than any other format in the AI education niche. Saves are the highest-signal engagement on Instagram — the algorithm treats them as "I'm coming back to this."

---

## Best Use Cases

- Free AI certifications (IBM, Google, DeepLearning.AI, Microsoft, Coursera free tier)
- AI tools for a specific role (developers / designers / content creators / marketers)
- AI courses ranked by quality, not popularity
- Resources to build a specific skill (prompt engineering, AI agents, fine-tuning)
- LinkedIn profile sections — AI certifications to add
- GitHub repos every AI learner should know
- AI newsletters / YouTube channels / communities

**Avoid for:** Vague "best resources" lists without specific, verified quality. Lists of more than 5 items (too long for a reel — use carousel format instead). Paid resources without clearly explaining the value proposition of the price.

## Hook Structures

**Hook Type A — The Roast Hook (canonical AI_SNIPP style, proven by REEL.md)**
```
"Bete... agar sahi se padha hota na, 
toh aaj ye haal nahi hota.

Tum logon ko bhi ye din na dekhna pade,
isliye laya hoon [N] AI [resources]

```

**Hook Type B — The Shortage Hook**
```
"Interview mein AI experience maanga — 
resume mein kuch tha nahi.

Ye [N] certifications fix kar deti hain ye problem."
```

**Hook Type C — The Value Proposition Hook**
```
"Ye [N] AI certifications free hain, 
LinkedIn pe dikhti hain, 
aur recruiters notice karte hain.

Seedhi baat."
```

**Hook Type D — The Comparison Hook**
```
"Paid AI courses mein ₹50,000 lagaane se pehle —
ye [N] free certifications dekho.
Same value. Zero cost."
```

**Selection rule:** Type A when the audience has been avoiding something they should be doing (classic "bete" energy). Type B when targeting interview anxiety specifically. Type C for practical-minded audience. Type D when countering paid course content from competitors.

## Story Structure

```
[0-8s]   HOOK     → Full screen creator. Roast/problem hook. Why this list matters.
[8-16s]  ITEM 1   → First resource: creator facecam + resource screen. 8 seconds.
[16-24s] ITEM 2   → Second resource: same structure. 8 seconds.
[24-30s] ITEM 3   → Third resource: same + CTA woven in. 8 seconds.
[30-35s] CTA      → Optional standalone CTA clip. Or end Item 3 clip with CTA.
```

**For 4-5 item lists:** Keep each item beat to 5-6 seconds instead of 8. Faster pace signals the list has density — which increases perceived value.

**The item beat structure (per item):**
```
[Second 1-2]: Creator names the resource. Says the name clearly.
[Second 2-6]: Screen shows the resource interface/certificate/UI.
[Second 6-8]: Creator delivers the one-sentence value proposition for this resource.
```

## Scene Structure

| Clip | Block | Duration | Content |
|---|---|---|---|
| Clip 1 | Block 1 — HOOK (SARCASTIC ROAST) | 8s | Full screen creator, hook delivery, problem named |
| Clip 2 | Block 2 — FACECAM + Resource 1 | 8s | Creator facecam + Resource 1 screen/UI |
| Clip 3 | Block 2 — FACECAM + Resource 2 | 8s | Creator facecam + Resource 2 screen/UI |
| Clip 4 | Block 2 — FACECAM + Resource 3 + CTA | 8s | Creator facecam + Resource 3 + CTA woven at end |

**Alternatively (clean separation):**
Clips 2, 3, 4 as resource beats + Clip 5 as standalone CTA = 5 clips total for a premium list.

## Character Usage

**Clip 1 (Hook):** SARCASTIC ROAST direction — this is the canonical "bete..." energy from AI_SNIPP_REEL.md. Playful exasperation. Creator looks directly at camera with slight head shake — the "I can't believe you don't know this" look that is warm, not mean.

**Clips 2-4 (Resource beats):** Creator as facecam overlay, bottom-right. Engagement mode — looking at the main content area (the resource screen) and then returning to camera for the value proposition line. The character in these clips is in "trusted guide" mode: not performing excitement, demonstrating informed enthusiasm.

**Item 3 + CTA beat:** The last resource clip must end with the CTA woven naturally. Expression shifts from informative to warm/direct as the CTA lands.

**Hand gesture rule from REEL.md:** "Do not count with fingers. Do not raise three fingers. Do not make number gestures while speaking." This rule exists because it was specifically noted in the canonical reel. Natural gestures only.

## Camera Language

| Clip | Camera | Movement | Notes |
|---|---|---|---|
| Hook | Slightly below eye (authority) | Push-in (3% over 8s) | The full-screen hook earns authority for the list |
| Resource 1 | Facecam 25% bottom-right | None | Resource screen must be crisp and legible |
| Resource 2 | Facecam 25% bottom-right | None | Creator looks at screen, then camera for value line |
| Resource 3 + CTA | Facecam, then pull to full screen | Brief static at end | Transition to full screen for CTA delivery |

**Resource screen requirement:** All resource screens must be realistic, professional dark-mode UIs. Certificate mockups must look like actual certificates (not cartoons). Course dashboards must show realistic course content and structure.

---

## Editing Style

- **Item numbering:** When each resource is introduced, a large number appears briefly: "01" → "02" → "03". Clean, minimal, white. Pops in and fades within 0.8 seconds. This gives the list visual structure.
- **Resource name typography:** When creator says the resource name, it appears as a bold text overlay — same typography as the caption system. Large, white, 1.5 seconds.
- **Certificate visuals:** When showing a certification, zoom in briefly (0.3s, 8%) to the certificate image — the close-up makes it look impressive and real.
- **Transition between items:** Quick cut, not a fancy transition. Speed-ram between Clips 2/3/4 is acceptable — creates pace and signals "there's more coming."
- **End card:** After the final CTA, a brief (1-second) end card showing all three resources listed. White text on dark background. This is added in post via Gemini Omni.

---

## Audio Style

- **Hook:** Music fades in under the AGITATE beat within the hook clip (not at the very start).
- **Resource beats:** Slightly more upbeat/educational music under the list items. Not loud — the voice and the resource screen are both carrying information.
- **Item transition SFX:** A soft "pop" or "swoosh" between each resource clip. Signals list progression.
- **Certification reveal SFX:** Soft positive chime when the certificate or resource is shown. Subtle — not a victory sound.
- **Music style:** Moderately upbeat, educational feel. Warmer than the news formula, less cinematic than Hollywood VFX.

## Flow Generation Strategy

**Resource screen quality is the critical variable.** A great hook with a bad-looking resource screen tanks the formula. Each resource screen must be described precisely in the Flow prompt.

**Clip 1:** Block 1 — HOOK SCENE. SARCASTIC ROAST direction. Standard dark studio. Push-in camera movement. "बेटे..." energy — warm exasperation, not meanness.

**Clip 2 (Resource 1):** Block 2 — FACECAM OVERLAY. Main content: describe the resource screen specifically. Example for IBM AI Fundamentals: "IBM Learning platform interface. Dark-mode dashboard. Shows 'IBM AI Fundamentals' course with progress bar at 100%, completion date, and a digital badge preview. The IBM logo is visible. Course modules are listed below. Professional educational interface."

**Clip 3 (Resource 2):** Block 2 — FACECAM OVERLAY + CONTINUATION note. Different resource screen. Creator's expression shifts slightly between items — engaged, not static. Add: "Creator occasionally glances toward the main content screen and back to camera during delivery. Nods when mentioning the key value of each resource."

**Clip 4 (Resource 3 + CTA):** Block 2 transitioning to Block 5. The clip begins with the resource screen (facecam overlay) and then the final 3 seconds shift to full-screen creator for the CTA. This can be achieved as one 8-second clip or two separate clips (preferred for cleaner execution).
