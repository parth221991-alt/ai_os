# F04 — HIDDEN TOOL

**Formula ID:** F04  
**Type:** Tool Discovery / Announcement  
**Reel Duration:** 25–30 seconds  
**Production Complexity:** Medium — requires tool UI demo clip  
**Repeat Frequency:** High — use 2× per week, different tool each time  
**Primary CTA:** DM trigger (TOOLS) — this is the highest-converting formula for list-building  
**Derived from:** APIFY signature CTA "DM me the word TOOLS and I'll send you the full list" + competitor hook analysis "tool demo with wow moment"

---

## Purpose

Positions @ai_snipp as the source that finds AI tools before everyone else. The formula combines discovery energy (the insider who found something) with immediate demonstration (prove it works) and an exclusive list offer (DM for more). Every Hidden Tool reel simultaneously builds credibility AND the DM subscriber list.

**Why it works:** The audience's deepest desire is knowing tools others don't know. This formula satisfies that desire immediately and promises more of the same. The DM trigger creates a low-friction first action — lower barrier than "follow" and higher value signal.

**Psychological mechanism:** Discovery excitement → "they found it for me" → immediate value from demo → FOMO on the full list → DM.

---

## Best Use Cases

- New AI tools with <6 months of Indian audience awareness
- Underrated tools that serve specific professional functions (developers, content creators, designers)
- Free alternatives to expensive paid tools (Canva → Ideogram, Photoshop → Adobe Firefly → free alternative)
- Niche tools with specific high-value use cases (AI for code review, AI for resume, AI for SQL)
- Updated features in existing tools that most users don't know about
- Indian-specific tools or tools with strong India use cases
- Claude API features that developers aren't using

**Avoid for:** Tools that are already widely known in the Indian tech community (ChatGPT basics, Midjourney for the third time). Avoid tools you haven't personally verified work as advertised.

## Hook Structures

**Hook Type A — The Discovery Announcement (most used)**
```
"Ek AI tool mila — aur seriously, 
isko dekh ke mujhe khud hairan ho gaya.
[Tool name] — suno kya karta hai ye."
```

**Hook Type B — The Problem-Solution Opener**
```
"Tum [X kaam ke liye] kitna time lagate ho?
[Pause]
Ye tool hai — [tool name]. 
Ye usi kaam ko [time/effort specification] mein kar deta hai."
```

**Hook Type C — The "Nobody Told You" Hook**
```
"Ye tool exist karta hai aur kisi ne 
tech Twitter pe bhi mention nahi kiya.
Main bol raha hoon — [tool name]."
```

**Hook Type D — The Roast Hook (targets existing behavior)**
```
"Bete... abhi bhi manually [task] kar rahe ho?
[Tool name] free hai. Download karo."
```

**Hook Type E — The Price Shock Hook**
```
"[Expensive tool] ke ₹[price] lagte hain.
[New tool] same kaam — free.
Aur iske baare mein koi nahi jaanta."
```

**Selection rule:** Type A for genuinely new tools. Type B for tools that solve a well-known frustration. Type C for tools that have zero awareness. Type D for tools that replace manual/tedious tasks. Type E for free alternatives to paid tools.

## Story Structure

```
[0-3s]   HOOK       → Name the tool. Create curiosity. One specific claim.
[3-8s]   AGITATE    → The problem this tool solves. Make the pain real.
[8-20s]  INSIGHT    → The tool in action. Live demo. The wow moment.
[20-26s] PROOF      → Result. Before/after. Time saved. Output quality.
[26-30s] CTA        → "DM karo TOOLS — main full list bhej dunga."
```

**The INSIGHT beat is the entire value of this formula.** If the demo is weak or unclear, the formula fails. The demo must show ONE specific thing the tool does and show it clearly. Don't try to cover all features in 12 seconds. Show the single best feature.

## Scene Structure

| Clip | Block | Duration | Content |
|---|---|---|---|
| Clip 1 | Block 1 — HOOK (DISCOVERY energy) | 8s | Creator full screen, tool announced, problem named |
| Clip 2 | Block 3 — TOOL DEMO Phase 1 | 5s | Creator at desk, loading/opening the tool |
| Clip 3 | Block 3 — TOOL DEMO Phase 2 (Result Reveal) | 8s | Tool output / demo + creator facecam |
| Clip 4 | Block 5 — CTA | 5s | DM trigger delivery, warm, direct |

**Optional Clip 2b (between Clips 2 and 3):** A 3-second "here's what I typed" clip showing the input before the output. This works well for AI writing or image tools where the prompt/input is part of the story.

---

## Character Usage

**Clip 1:** INSIDER SECRET acting direction (not SARCASTIC ROAST — discovery tools have genuine excitement energy, not irony). Slight conspiratorial quality — "main tumhe kuch bata raha hoon jo aur log nahi jaante" body language. Head tilts slightly toward camera. The delivery is fast but not rushed — there's pleasure in the reveal.

**Clip 2:** Desk setup. Purposeful, efficient movements. Opens the tool the way someone who uses it daily would — not performing discovery, demonstrating familiarity.

**Clip 3:** Creator as facecam overlay, bottom-right. The expression during the result: impressed nod. The character never performs shock — but this formula allows slightly more visible reaction than others because the discovery energy is genuine.

**Clip 4:** Standard warm CTA close. The energy has come down from discovery excitement to warm directness. "I found this. Here's how you get more like it."

## Editing Style

- **Tool name typography:** When the tool is first mentioned, its name appears as a text overlay — large, white, bold. Appears simultaneously with creator saying the name. 1.5 seconds on screen. Clean pop-in animation, no slide.
- **Demo screen treatment:** When the tool interface appears (Clip 3), add a brief zoom-in (0.3s, 5%) to "push" viewer into the interface. This creates the sensation of leaning in.
- **Result animation:** When the tool output appears (the "wow moment"), add a subtle screen flash or pop-in. Short riser SFX (0.5s).
- **Comparison treatment:** If showing before/after or alternative tool comparison, use a clean split-screen or sequential cut with the "vs" text card in between.
- **Tool website/URL:** Can appear briefly as a text overlay at the end of the demo clip (2 seconds). Helps credibility and reduces friction for the viewer who wants to act immediately.

---

## Audio Style

- **Music:** Starts under the AGITATE beat. Upbeat-but-controlled electronic. This is the closest AI_SNIPP gets to "energetic" music — the discovery formula has the most positive energy.
- **SFX:** 
  - Tool name reveal: soft chime or subtle notification sound (signals "new thing found")
  - Demo interaction sounds: realistic UI sounds if tool has them; otherwise minimal click sounds
  - Result reveal: short rising tone + subtle impact
  - Before/after transitions: clean swipe
- **Voice:** Fastest delivery of any formula. The discovery excitement naturally accelerates speech. This is fine — it signals genuine enthusiasm without performance.

## Flow Generation Strategy

**Pre-production note:** Before generating any Flow clips, navigate to the tool and identify the single most impressive feature to demonstrate. This feature selection is 50% of the formula's success. Write a one-sentence description of the "wow moment" before starting Flow.

**Clip 1:** Block 1 — HOOK SCENE. INSIDER SECRET acting direction. Key addition: "The character's energy is discovery excitement — slightly elevated from standard delivery, as if sharing something he just found. Not performative excitement — genuine enthusiasm of someone who found a good thing and wants to share it."

**Clip 2:** Block 3 — TOOL DEMO Phase 1. Environment D (Desk). The tool should be described in detail in the prompt: its interface color scheme, what it shows, what action is happening. Generic "tool interface" prompts produce generic outputs.

**Clip 3:** Block 3 — TOOL DEMO Phase 2. Main content: the tool output. Must be described specifically: "The [TOOL NAME] interface showing [THE SPECIFIC OUTPUT]. Dark mode. Clean professional UI. The output is [describe what it shows]. This is the 'wow moment' — the tool has produced something impressive."

**Clip 4:** Block 5 — CTA. DM trigger. Eye level. Standard.
