# F06 — AI VS HUMAN

**Formula ID:** F06  
**Type:** Side-by-Side Comparison / Time-Compression Proof  
**Reel Duration:** 25–35 seconds  
**Production Complexity:** Medium-High — requires both "human" and "AI" demonstration elements  
**Repeat Frequency:** Medium — use 1–2× per week, different task each time  
**Primary CTA:** Save CTA (highest saves of any formula) + DM trigger  
**Derived from:** APIFY Week 3 "PROOF — Here's what's possible" + competitor analysis showing comparison content drives highest engagement

---

## Purpose

Demonstrates the productivity gap between working with AI and without it. The viewer sees the same task done in two ways — the human/manual way they currently do it, and the AI-assisted way they should be doing it. The delta (time, quality, effort) is the entire value proposition.

**Why it works:** Comparison content is the most shareable format in the productivity niche. When a viewer saves 3 hours per week, they want their colleagues and friends to also save 3 hours per week. AI vs Human content is the most-shared format in the Indian tech content space.

**Psychological mechanism:** Recognition of current (slow) workflow → visible proof of better alternative → "I do this manually..." → share with teammates → follow to learn more.

---

## Best Use Cases

- Writing tasks: manual vs Claude (cover letter, email, SQL query, documentation)
- Research tasks: Google research vs Perplexity AI
- Design tasks: manual Canva vs AI-generated graphics
- Code tasks: writing code manually vs Claude code completion
- Data tasks: manual Excel analysis vs AI-powered analysis
- Content creation: writing from scratch vs AI-assisted drafting
- Resume/LinkedIn: manual optimization vs AI-powered rewrite

**Avoid for:** Tasks where the human outcome is genuinely better (do not fake the comparison for content). If AI doesn't clearly win the comparison, don't use this formula for that topic.

## Hook Structures

**Hook Type A — The Time Reveal (most powerful)**
```
"Ye kaam mujhe 2 ghante lagta tha.
Claude ne ye kiya — dekho — 
[timer: 00:47]
47 second."
```

**Hook Type B — The Quality Reveal**
```
"Main likhta hoon toh aisa hota hai.
Claude likhta hai toh ye hota hai.
Main honestly embarrassed hoon."
```

**Hook Type C — The "I Still Did This Manually" Confession**
```
"Seedha bol raha hoon — 
main abhi bhi ye manually karta tha 2 mahine pehle.
Phir kisine dikhaya. Aur ab... dekho."
```

**Hook Type D — The Challenge Hook**
```
"Human vs AI. Same task. Same time limit.
Main time kar raha hoon.
Aap judge karo."
```

**Selection rule:** Type A for pure time-efficiency content. Type B for quality comparison (writing, design). Type C for relatable confession content (builds authenticity). Type D for high-energy comparison content.

## Story Structure

```
[0-3s]   HOOK       → Name the task. Name the claim (time/quality). Create the expectation.
[3-8s]   HUMAN WAY  → Show or describe the manual/human way briefly. Make the pain real.
[8-22s]  AI WAY     → Show the AI doing the task. The full demo. Real-time if possible.
[22-28s] COMPARISON → Show both results side by side OR state the delta explicitly.
[28-32s] CTA        → "Save karo" — this content is reference material.
```

**The HUMAN WAY beat is underused by most creators.** Most creators skip directly to "AI does it better." But the setup moment — 5 seconds of the manual/human way — is what makes the comparison emotionally land. Without showing the pain of the old way, the relief of the new way doesn't hit.

## Character Usage

**Clip 1 (Hook):** SHOCK VALUE direction — starts neutral, delivers the time/quality claim, slight raise of brow at the punchline. The delivery is underplayed — the statistic does the work, not the performance.

**Clip 2 (Human Way):** Storytelling camera (eye level). Creator speaks from personal experience: "Main ye manually karta tha..." Personal confession tone. Slight self-deprecating quality — the character is never above his audience.

**Clip 3 (AI Demo):** Desk setup. Creator types the AI prompt efficiently — not slowly for the camera, at actual working speed. This authenticity matters. The character should look like someone who has done this a hundred times.

**Clip 4 (Comparison/Output):** Facecam overlay or full screen comparison card. Expression: "Fark dikhta hai?" — the "I'm not making this up" look. Nod.

**Clip 5 (CTA):** Warm, direct. Eye level. "Save karo" energy — genuine, not pushy.

|---|---|---|---|

| CTA | Eye level | None | Static, warm |

## Editing Style

- **Timer element:** For time-compression content, add an animated timer graphic in the corner during the AI demo. Shows actual time passing. Nothing creates more urgency than a visible countdown.
- **Comparison treatment:** For quality comparisons — side-by-side split screen is the gold standard. Left side: "Human (manual)" with a slightly desaturated, cooler treatment. Right side: "AI (Claude/GPT)" with a warmer, cleaner treatment. The visual difference reinforces the quality difference.
- **Speed ramp:** The manual/human way can be slightly sped up (1.5×) and the AI way plays at normal speed. This creates the visual sensation of "this used to take forever."
- **Timestamp / Before-After text:** Add clean text overlays: "BEFORE: 2 hours" and "AFTER: 47 seconds." White text on dark background. Large and clear.
- **Output quality treatment:** When the AI output appears, pause for 1 second (no movement, no SFX) — let the viewer read it and absorb the quality. Then continue.

---

## Audio Style

- **Hook beat:** Higher-energy music than most formulas. This is a competition — the audio should match the competitive energy.
- **During AI demo:** Music at full level under the demo. SFX: keyboard typing sounds. The combination of typing sounds + music creates authentic "at work" atmosphere.
- **During comparison:** A clean "reveal" sound when the comparison is shown. Soft impact or a simple musical resolution — signals "and the winner is..."
- **Timer SFX:** If using a timer element, tick sounds under the AI demo (every 5 seconds). Builds tension and makes the time visible AND audible.

## Flow Generation Strategy

**Pre-production requirement:** Do the actual comparison yourself before generating any clips. Time it. Measure the quality difference. The numbers you quote must be real — the Indian tech audience will test these claims.

**The "Human Way" visual (Clip 2):** This does not need a complex Flow generation. This is the creator speaking about their own experience — storytelling camera, eye level, personal confession tone. Simple to generate.

**The "AI Way" clips (Clips 3-4):** These are the technically demanding clips. The AI tool interface must be realistic and show an impressive output. Describe it precisely: "Claude.ai interface showing a conversation. Creator's prompt visible at the bottom: '[THE EXACT PROMPT]'. The response above shows [describe the impressive output specifically]."

**Comparison visual:** For quality comparisons, consider generating this as a static graphic (not Flow) — side-by-side card with "Human" on the left (simple, smaller, less polished) and "AI Output" on the right (detailed, comprehensive, impressive). Use Canva or Figma for this element.
