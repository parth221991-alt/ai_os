# F01 — AI CHEAT CODE

**Formula ID:** F01  
**Type:** Tool Shortcut / Hidden Feature Reveal  
**Reel Duration:** 20–28 seconds  
**Production Complexity:** Low — single location, no complex compositing  
**Repeat Frequency:** High — use 2–3× per week with different topics  
**Primary CTA:** DM trigger ("DM karo TOOLS")  
**Derived from:** APIFY hook style "tool demo with wow moment" + AI_SNIPP_REEL.md lesson "Premium UI visuals outperform generic AI animations"

---

## Purpose

The fastest-converting formula in the AI content niche. Reveals one specific, actionable AI feature or trick that the audience didn't know existed. The value is delivered in under 30 seconds. The viewer leaves with something they can use immediately.

**Why it works:** The Hinglish tech audience is deeply practical. They don't want inspiration — they want shortcuts. This formula delivers the shortcut and nothing else. Every second of content earns its place.

**Psychological mechanism:** Knowledge gap → immediate closure → dopamine → share/save.

---

## Best Use Cases

- Hidden features inside Claude, ChatGPT, Gemini, Perplexity
- Prompt engineering tricks (single-line prompts that do dramatic things)
- Keyboard shortcuts or workflow hacks in AI tools
- Lesser-known AI tools that solve a specific problem
- API tricks or Claude Projects features
- Free alternatives to expensive AI tools
- Settings/configurations that 99% of users miss

**Avoid this formula for:** Conceptual content (what AI means), industry analysis, career advice, long tutorials.

## Hook Structures

All hooks must create a knowledge gap in 3 seconds or fewer. Choose based on content type:

**Hook Type A — The Percentage Hook (Insider Exclusivity)**
```
"[X]% log ye Claude feature use hi nahi karte — 
aur main unhe blame bhi nahi karta, kisi ne bataya hi nahi."
```

**Hook Type B — The Time Compression Hook**
```
"Jo kaam mujhe 45 minute lagta tha, 
Claude ki ye ek setting se 3 minute ho gaya."
```

**Hook Type C — The Discovery Hook (Conspiratorial)**
```
"Yaar, ye feature 6 mahine se exist kar raha hai 
aur mujhe aaj pata chala."
```

**Hook Type D — The Roast Hook (Bete... energy from AI_SNIPP_REEL.md)**
```
"Bete... agar tum abhi bhi ChatGPT ko normal chatbox 
ki tarah use kar rahe ho, toh sun."
```

**Hook Type E — The Result-First Hook**
```
"Dekho main kya bana diya — 
ek prompt. Bas ek prompt."
```

**Selection rule:** Use Type A for tool settings. Type B for workflow efficiency. Type C for new features. Type D for audience that is intermediate-level. Type E for visual content where the output IS the hook.

## Story Structure

```
[0-3s]   HOOK     → Knowledge gap. One line. Creator full screen.
[3-8s]   AGITATE  → Why this matters. What they're losing by not knowing this.
[8-20s]  INSIGHT  → The cheat code itself. Show it working. Creator + tool UI.
[20-25s] PROOF    → The result. One clear before/after or output demonstration.
[25-28s] CTA      → "DM karo TOOLS" or follow.
```

**The AGITATE beat is critical.** Most creators skip it and go directly from hook to insight. Agitate makes the viewer care. "Agar ye nahi pata, toh tum X ghante extra kaam kar rahe ho har roz" — this line earns the insight.

## Character Usage

**Clips 1 & 4:** Full screen, direct camera, confident. Hook uses SARCASTIC ROAST or INSIDER SECRET acting direction.

**Clip 2:** Desk setup. Creator leaning toward off-screen monitor, one hand near keyboard. Expression: focused, mischievous — "watch what happens."

**Clip 3:** Creator as facecam overlay (bottom-right). Expression: slow nod, contained smirk. "I told you." The character never over-celebrates. The restraint is the brand.

**Voice direction (from Voice 3.1/6.3 files):** Fast, confident, loud-but-not-shouting. In the AGITATE beat, pace slows briefly for one line — then accelerates again into the insight.

## Editing Style (Gemini Omni directions)

- **Caption style:** Kinetic typography — key tech terms in larger, bolder text. "Claude", "Projects", "prompt" get typographic emphasis.
- **Pattern interrupt timing:** Visual change every 1.5–2 seconds in the INSIGHT beat. Shorter intervals = higher retention.
- **SFX on each cut:** Subtle whoosh or swipe between Clip 1 → Clip 2. Clean impact sound on the result reveal.
- **Tool UI treatment:** When the tool screen appears, brief zoom-in (0.3s, 5%) to "push" the viewer into the interface.
- **Creator face during tool demo:** Facecam must have amber glow border. Face sharp. Tool UI underneath must be in dark mode.
- **Background text:** During the INSIGHT beat, a large semi-transparent text of the tool name can appear behind the creator in the facecam — e.g., "CLAUDE PROJECTS" at 40% opacity with subject masking.

---

## Audio Style

- **Hook:** No music under the first 3 seconds — let the hook land on silence. Music fades in under the AGITATE beat.
- **Music:** Medium-energy cinematic/electronic instrumental. Forward-moving. Not aggressive. Keeps pace with speech.
- **SFX palette:** 
  - Cut transitions: quick air swipe
  - Result reveal: short rising tone/riser (0.5s)
  - Each tool-click moment: subtle click/tap sound
  - Key word emphasis: short soft impact
- **Voice:** Original audio primary. Music and SFX are background. Never compete with voice.

## CTA Structure

**Primary (used 70% of the time):**
```
"Agar aur aisi AI cheat codes chahiye, toh DM karo 'TOOLS' — 
main personally list bhej dunga."
```

**Secondary — Save CTA (when content is reference-type):**
```
"Isko save kar lo — kal kaam aayega."
```

**Tertiary — Comment engagement (when topic has debate potential):**
```
"Ye feature pata tha tumhe? Comment mein batao — 
'haan' ya 'nahin' — genuinely jaanna hai."
```

**CTA selection rule:** If the content is a tool list → DM trigger. If it's a single tip → Save CTA. If it's controversial or surprising → Comment engagement.

## Flow Generation Strategy

**Session order:** Generate Hook clip first. Verify against 5-point consistency check. Then generate Tool Demo clips. CTA clip last (highest drift risk — always generate immediately after a passing hook clip in the same session).

**Clip 1 (Hook):** Block 1 — HOOK SCENE + Environment A (Dark Studio). Acting direction: SARCASTIC ROAST or INSIDER SECRET. Attach Character.png.

**Clip 2 (Tool Demo Phase 1):** Block 3 — TOOL DEMO Phase 1. Environment D (Desk Setup). Creator left of frame, facing screen right. Add CONTINUATION note.

**Clip 3 (Tool Demo Phase 2):** Block 2 — FACECAM OVERLAY. Main content: describe the specific tool UI in detail. Add CONTINUATION note. Key: the tool UI description must be precise — generic descriptions produce generic outputs.

**Clip 4 (CTA):** Block 5 — CTA SCENE. Environment A with dimmed monitors. Eye level. DM trigger delivery. Add CONTINUATION note.

**Consistency risk for this formula:** Medium. The tool demo clips (2 & 3) use desk/partial-body angles which are harder to maintain than full-screen authority shots. Prioritize hair and glasses check on these clips specifically.
