# F09 — THREE STEP SYSTEM

**Formula ID:** F09  
**Type:** Tutorial / Actionable Framework  
**Reel Duration:** 28–35 seconds  
**Production Complexity:** Medium — requires tight script, optional screen demo  
**Repeat Frequency:** High — use 2× per week, different system/workflow each time  
**Primary CTA:** Save CTA (this is the highest save-rate formula alongside F06)  
**Derived from:** APIFY Week 2 "EDUCATION" theme + Week 4 "ENGAGEMENT" + competitor analysis showing step-by-step tutorial content drives highest save rates across all formats

---

## Purpose

Delivers a complete, actionable system in three steps. Not three tips — a system. The distinction matters: tips are isolated, a system is a workflow that the viewer can run end-to-end. The viewer finishes this reel holding a complete mental model they can immediately act on.

**Why it works:** The Indian tech audience is action-oriented. They don't save content to think about later — they save it to run later. A three-step system gives them a complete unit of executable knowledge. The number three is psychologically optimal: two feels thin, four feels like work, three feels achievable.

**Psychological mechanism:** "I've been doing this inefficiently" → clear, numbered steps → "I can do this immediately" → save for future reference → follow for more systems.

**The creator's role in this formula:** The trusted senior colleague who has already figured out the most efficient path and is giving it to you. Not the teacher explaining the subject — the friend who solved it and is sharing the shortcut.

---

## Best Use Cases

- AI-powered workflow systems (research → draft → publish in 3 AI steps)
- Code review system using AI (read → question → improve)
- Interview preparation system (resume audit → mock interview → follow-up optimization)
- Prompt engineering system (role → task → constraints → output)
- Learning a new skill system (overview → hands-on → feedback loop)
- Daily AI productivity routine (morning brief → deep work → review)
- Content creation pipeline (idea → structure → execution with AI)
- Trading/finance research system (macro → sector → stock with AI tools)
- Resume + LinkedIn + portfolio system (3 steps, all AI-assisted)

**Avoid for:** Processes that genuinely require more than 3 steps to be useful (do not force into 3). Processes where the viewer cannot start within 24 hours. Highly technical systems requiring prerequisites the average viewer won't have.

## Hook Structures

**Hook Type A — The System Promise (most used)**
```
"Mera [task] ka exact 3-step system —
jo mujhe [time] bachata hai har [week/day/project].
Note down karo."
```

**Hook Type B — The "I Used to Do It Wrong" Hook**
```
"Pehle [task] ke liye mujhe [old method] karna padta tha.
Ab ye 3 steps hain. Poori process.
Step 1 se shuru karte hain."
```

**Hook Type C — The Credential Hook**
```
"[N] baar ye karne ke baad mujhe pata chala —
ye 3 steps hi kaam aate hain.
Baaki sab noise hai."
```

**Hook Type D — The "Save Before You Watch" Hook**
```
"Ye reel save karo pehle.
3 steps hain — baad mein kaam aayenge.
Phir dekho."
```

**Hook Type E — The Problem-First Hook**
```
"Tum [task] ke liye poora din lagate ho.
3 steps mein ho sakta hai.
Main dikhata hoon."
```

**Selection rule:** Type A for proven personal systems the creator actually uses. Type B for systems that replace an obviously inefficient old approach. Type C when the repetition/experience angle is the credibility. Type D for dense, reference-heavy systems (preemptive save instruction). Type E for workflow efficiency content.

## Story Structure

```
[0-5s]   HOOK         → Name the system. State the benefit. Create the expectation of 3 steps.
[5-10s]  STEP 1       → Clear name + what it does + how to do it (2-3 sentences max).
[10-18s] STEP 2       → Clear name + tool or method + brief demo or example if possible.
[18-25s] STEP 3       → Clear name + the culminating action + the outcome it produces.
[25-30s] SYSTEM RECAP → One line that binds all 3 steps together. The "this is why it works."
[30-35s] CTA          → "Save karo — ye system wahan milega."
```

**Each step must have: a name, an action, and a result.** "Step 1: Research" is incomplete. "Step 1: Research — Use Perplexity to pull 5 sources in 3 minutes. Output: a source list with summaries." is a step.

**The System Recap beat is underused.** After 3 steps, one binding sentence that explains the system's logic is what elevates this from a list to a framework. Example: "Teen steps mein ye kaam karta hai kyunki pehle scope clear hoti hai, phir content ban jata hai, aur aakhri step quality lock kar deta hai."

## Scene Structure

| Clip | Block | Duration | Content |
|---|---|---|---|
| Clip 1 | Block 1 — HOOK (system delivery) | 8s | Creator full screen, system introduced, benefit stated |
| Clip 2 | Block 6 — STORYTELLING (Step 1) | 8s | Creator eye level, Step 1 delivered — clear and specific |
| Clip 3 | Block 3 or 6 — Step 2 (with optional demo) | 8s | Step 2 — if tool is involved, 3s of tool UI, 5s creator |
| Clip 4 | Block 2 or 6 — Step 3 + Recap | 8s | Step 3 delivered, system recap, transition to CTA |
| Clip 5 | Block 5 — CTA | 5s | Save CTA delivery, warm, direct |

**Screen demo rule:** If a step involves a specific tool, show it for 3-5 seconds. If the step is conceptual (a mindset, a framework), keep the creator full-screen. Do not force screen demos when they add nothing.

**Numbering visual:** Each step should appear with a visual number: "01" → "02" → "03". Large, white, clean. Same treatment as F05. Pops in for 0.8s as the creator says the step name.

---

## Character Usage

**Clip 1:** DELIVERY mode — not sarcastic, not urgent. The character is sharing a system, which has a different energy: organized, confident, generous. The body language is "I have something useful and I'm giving it to you clearly." Slightly more upright than storytelling mode, slightly less intense than urgency mode.

**Clip 2-4 (Steps):** Eye level. Peer-to-peer explanation. The character in these clips is talking to one person — not performing for a crowd. The tone is collegial: "Ye jo step hai — ye mujhe personally bahut kaam aaya." First-person use claims add credibility.

**Step with demo (Clip 3 typically):** Creator as facecam overlay while the tool is the main content. Nods while looking at the tool screen. Returns to camera for the "why this step works" line.

**Clip 5 (CTA):** Warm, satisfied delivery. The character has just shared something genuinely useful and knows it. Not self-congratulatory — just direct and warm. "Save karo — ye sab wahan rahega."

## Editing Style

- **Step numbering:** "01" → "02" → "03" appear as large text overlays. White, bold, no background. Pop in for 0.8 seconds at the start of each step beat. Fade out. This is the visual spine of the formula.
- **Step name typography:** When the step name is said, it appears simultaneously as a bold text overlay. Slightly smaller than the number. Same color. On screen for 1.5-2 seconds.
- **Pace markers:** Between Step 1/2 and Step 2/3, a brief (0.3s) visual cut or pattern interrupt. A quick flash, a wipe — something that signals "that was one step, here comes the next." This keeps the viewer aware of progress through the list.
- **System recap treatment:** The recap sentence, when delivered, can appear as text on screen simultaneously. Clean, centered, white. This is the pullquote treatment — the single most shareable/saveable text element in the reel.
- **Color grade:** Standard (@ai_snipp palette). No special grade — this is the warmest, most collegial formula. The color treatment should feel like a conversation.
- **End card (optional):** A 1-2 second end card showing all 3 steps listed. "Step 1: [Name] → Step 2: [Name] → Step 3: [Name]". White on dark. This is the image the viewer saves as a reference.

---

## Audio Style

- **Music:** Starts under the HOOK beat. Moderate, slightly upbeat but not aggressive. This is the educational formula — the music should feel supportive, not distracting. Think "focused study session" rather than "discovery excitement."
- **Step transition SFX:** A soft click or pop between each step. Not the same heavy impact used in F06/F07 — lighter, to signal progress without drama.
- **Recap SFX:** A subtle resolution sound when the system recap lands. Something that signals "this is the insight" — a soft positive chime works.
- **Creator voice:** Normal to slightly elevated pace. The steps should be delivered with deliberate clarity — the viewer needs to absorb each step. Don't rush steps for the sake of energy.
- **Step name emphasis:** Creator voice should give a slightly heavier emphasis to the step name itself. "Step 1 hai RESEARCH — iske liye Perplexity use karo." The name gets the weight.

## Flow Generation Strategy

**Script tightness is the critical variable.** Each step must be pre-written to exactly fit its 8-second clip. Do not generate Flow prompts before the script is finalized — loose steps produce loose clips.

**Pre-production requirement:** Write all 3 steps fully before generating any clips:
- Step name (2-3 words)
- Action sentence (what the viewer does)
- Tool/method (specific — not "an AI tool")
- Output description (what they get)
- Creator's one-line value statement ("Ye step critical hai kyunki...")

**Clip 1:** Block 1 — HOOK SCENE. Acting direction: SYSTEM DELIVERY (not SARCASTIC ROAST, not URGENCY). Key addition: "The character's energy is organized confidence — the body language of someone who has run this system many times and knows it works. Not performing excitement. Demonstrating competence."

**Clip 2 (Step 1):** Block 6 — STORYTELLING. Eye level. Natural gesture — the step name gets a hand movement (not finger counting — perhaps an open-hand gesture forward, the 'here it is' motion). The character looks directly at camera for the action sentence, briefly off-camera for the tool/method name (as if recalling from memory rather than reciting).

**Clip 3 (Step 2):** Block 6 OR Block 3 if demo. If using a demo: describe the tool UI specifically. "The [TOOL NAME] interface showing [the exact state during Step 2 — what's been typed, what result has appeared]. Creator facecam bottom-right, pointing toward the main content area." If no demo: Block 6, same natural delivery as Clip 2.

**Clip 4 (Step 3 + Recap):** Block 6 transitioning to Block 5 energy for the recap. The system recap is a distinct beat — creator's pace slows slightly, eyes may narrow slightly (the "this is the real insight" look). This beat is the closest the character gets to "wise mentor" mode.

**Clip 5 (CTA):** Block 5. Save CTA. Warm. Standard.
