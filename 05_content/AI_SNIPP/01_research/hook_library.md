# Hook Library
## AI_SNIPP Research & Intelligence System

*A hook is the single decision point. Viewer scrolls past or stops — decided in 1 second. This library contains every proven hook pattern available to @ai_snipp, adapted for Hinglish delivery and the Indian AI audience. Every hook here is sourced from real comment language, proven formulas, or direct extraction from the canonical reel (AI_SNIPP_REEL.md).*

*Rule: The first 3 words of a hook must create tension or curiosity without needing any prior context. If the first 3 words could appear at the start of any other reel, the hook is too generic.*

---

## HOOK ANATOMY

Every effective hook has three components:

```
[INTERRUPT] + [TENSION] + [PROMISE]

INTERRUPT = forces the eye/ear to stop (unexpected word, contrast, emotional trigger)
TENSION   = creates a gap the viewer needs to close (curiosity, fear, confusion)
PROMISE   = implicit or explicit reward for watching (learning, saving, fixing)
```

**Examples:**
- "Bete..." [INTERRUPT: unusual opener] + "Claude use karte ho aur system prompt nahi jaante?" [TENSION: viewer might be making this mistake] + "80% power waste kar rahe ho." [PROMISE: fix this = immediate gain]
- "Breaking:" [INTERRUPT: news signal] + "Yeh AI tool 6 months mein band ho jaayega." [TENSION: fear of missing window] + [Implicit PROMISE: watch to know what to do]

## HOOK CATEGORIES

### CATEGORY 1 — ROAST HOOKS ("Bete..." Pattern)

**Pattern:** Warm, mock-parental exasperation. "I can't believe you don't know this" energy delivered with genuine affection — not contempt.

**When to use:** F01 (Cheat Code), F04 (Hidden Tool), F05 (Career List), F09 (System). Best for topics where the audience should already know this — the roast makes the ignorance feel relatable, not shameful.

**Tone:** SARCASTIC ROAST acting direction. Steepled hands. Slight head shake. Half-smile that says "I'm roasting you but I genuinely want to help."

**Formula:**
```
"Bete... [observation of what the viewer is missing or doing wrong]. [Optional: consequence]. [Setup for the value to follow.]"
```

**AI_SNIPP Adapted Examples:**

1. "Bete... Claude use karte ho roz, aur system prompt nahi jaante? 80% power waste kar rahe ho."
   *(From EX01 — F01: AI Cheat Code)*

2. "Bete... agr sahi se padha hota na, to aaj yeh haal nahi hota. Tumhe bhi yeh din na dekhna pade, isliye laya hun 3 AI certifications."
   *(From AI_SNIPP_REEL_011 — F05: Career List)*

3. "Bete... ChatGPT pe rehte ho aur Claude ka Projects feature nahi jaante? Abhi chhod ke dekho farak."

4. "Bete... AI tools dhundh rahe ho ProductHunt pe? Main bata deta hun woh jo 90% log miss karte hain."

5. "Bete... resume mein 'AI skills' likha hai par prompt engineering nahi aata? Interview mein pata chal jaayega."

6. "Bete... ek hi Claude conversation mein 50 messages waste kar rahe ho tab bhi kuch nahi nikla? Yeh prompt structure dekho."

**Hook intensity:** Medium — warm enough to not alienate, sharp enough to stop the scroll.
**Best performing for:** F05, F01, F04
**Avoid for:** F07 (News Flash — wrong energy), F02 (Future Shock — too light for career urgency)

### CATEGORY 2 — URGENCY HOOKS ("Breaking" / "X mahine baad band")

**Pattern:** Creates genuine time pressure. The window is closing, or something just happened. NOT manufactured urgency — there must be a real reason this is time-sensitive.

**When to use:** F07 (News Flash), F02 (Future Shock). Use sparingly — if everything is urgent, nothing is urgent.

**Tone:** URGENCY acting direction. Slightly wider eyes. Forward lean. Faster-than-standard delivery opening.

**Formula A — Breaking News:**
```
"Breaking: [what just happened]. [Why this affects the viewer directly.]"
```

**Formula B — Closing Window:**
```
"[Timeframe] mein [X] band/change ho jaayega. [What the viewer needs to do before then.]"
```

**Formula C — Threshold:**
```
"[Company/tool/platform] ne [change] announce kiya. [Indian-specific implication in one line.]"
```

**AI_SNIPP Adapted Examples:**

1. "Breaking: Anthropic ne Claude 4 launch kiya. Yeh 5 changes tumhare kaam mein seedha asar dalenge."
   *(From EX02 — F07: News Flash)*

2. "OpenAI ne free tier ki limits cut ki. Claude users ke liye yeh actually good news hai — sunno kyun."

3. "Google ne NotebookLM ka Hindi support launch kiya. Indian creators ke liye yeh game-changer hai."

4. "2026 tak AI coding assistants har developer ke workflow mein rahenge. Abhi jo nahi seekha — 18 mahine baad pachtayega."

5. "Gemini 2.5 Flash free ho gayi. In 3 tools ko aaj replace kar sakte ho."

6. "Razorpay ne AI payments feature launch kiya. Indian developers ke liye yeh matlab hai..."

**Hook intensity:** High — generates genuine stop energy. Don't overuse.
**Overuse signal:** If you use urgency 3+ times in a week, the next urgency hook loses 50% of its power.
**Best for:** F07, F02
**Avoid for:** F01, F05 (wrong register)

### CATEGORY 3 — DISCOVERY HOOKS ("Maine dhundha...")

**Pattern:** Creator positions as the scout who found something before the audience did. Creates parasocial FOMO — "if this person is showing me this, I should pay attention."

**When to use:** F04 (Hidden Tool), F01 (Cheat Code — when discovery framing fits), F03 (Hollywood VFX — output reveal).

**Tone:** INSIDER SECRET acting direction. Conspiratorial head tilt. Lowered voice energy (even if text — the delivery implies "just between us").

**Formula A — Personal Discovery:**
```
"Maine [X] dhundha/discover kiya jo [audience] ke liye [benefit]. [Hindi audience don't even know this exists yet.]"
```

**Formula B — Source Reveal:**
```
"[X] tool kahan milti hai jisko 90% developers nahi jaante? [Location/name of the hidden source.]"
```

**Formula C — Accidental Discovery:**
```
"[Task] kar raha tha jab [accidental discovery]. Yeh feature sirf [audience segment] ke liye hai."
```

**AI_SNIPP Adapted Examples:**

1. "Yeh AI tool dhundhi jo aaj tak ProductHunt pe nahi aayi. Indian developers ke liye free hai."

2. "Claude mein ek hidden feature hai jo 3 paid tools ki jagah le sakta hai. Settings mein hai — koi nahi batata."

3. "Is tool ka naam nahi sunna hoga — 6 months pehle launch hui thi. Aaj free plan aayi hai Indian users ke liye."

4. "Hugging Face par ek repo hai jo 47,000 stars le chuki hai. No one in India is talking about this."

5. "YouTube pe AI tutorials dhundh raha tha — mujhe yeh GitHub repo mili jisko 200 Indian developers already follow kar rahe hain."

6. "Anthropic API ka yeh parameter sirf documentation mein hai — kisi video mein nahi dikha."

**Hook intensity:** Medium-high — creates curiosity without alarm.
**Best for:** F04, F01, F03
**Avoid for:** F07 (too personal for news), F02 (too casual for urgency)

### CATEGORY 4 — STATS HOOKS ("X% log...")

**Pattern:** Leads with a statistic that makes the viewer feel they're either on the wrong side of the data, or about to be. The stat must be real and specific — Indian AI audiences verify claims.

**When to use:** F06 (AI vs Human), F02 (Future Shock), F07 (News Flash with data angle), F08 (Transformation with time stat).

**Tone:** Slightly elevated, matter-of-fact. Not breathless. The creator presents the stat like a senior colleague reading off a report — knows it, owns it, will explain it.

**Formula A — Percentage Exclusion:**
```
"[X]% [audience] [action/mistake/fact]. Aap kahan hain?"
```

**Formula B — Time Compression:**
```
"[Task] mein [old time] lagte hain. AI se [new time]. [The gap] fark kya hai?"
```

**Formula C — Market/Career Data:**
```
"[Year] mein [AI stat]. [Indian-specific interpretation in one line]."
```

**AI_SNIPP Adapted Examples:**

1. "45 minute ka kaam — AI se 8 minute mein. Yeh mere actual test ka result hai."

2. "78% Indian developers abhi bhi ChatGPT pe hain. Claude ke yeh 3 features unhe nahi pata."

3. "AI se banaye reels ko 3x zyada saves milti hain. Yeh mera 90-day data hai."

4. "Prompt engineering jobs ki demand 2x ho gayi — 6 mahine mein. Indian market ka data hai yeh."

5. "1 cold email — 2 minute mein AI se. Manual mein 25 minute. Reply rate fark bhi dikhata hun."

6. "Gemini 2.5 Flash ne coding benchmark mein GPT-4 ko beat kiya — 73% vs 67%. Developer ke liye kya matlab hai?"

**Hook intensity:** Medium — data-driven, credible, not emotional.
**Best for:** F06, F02, F07 (data angle)
**Avoid for:** F05 (too transactional for career advice), F10 (too impersonal for community response)

### CATEGORY 5 — "NOBODY TELLS YOU" HOOKS

**Pattern:** Positions the creator as the honest insider who breaks silence on something the industry/creator community isn't talking about. Creates exclusive insider identity — "only @ai_snipp tells you this."

**When to use:** F01 (Cheat Code), F04 (Hidden Tool), F09 (System — when the system contradicts conventional advice).

**Tone:** Conspiratorial warmth. Slight lean-in. "I'm about to break ranks" energy without being dramatic.

**Formula A — Information Gap:**
```
"Koi nahi batata [specific fact/feature/trick]. Main bata raha hun — [the hidden thing]."
```

**Formula B — Deliberate Hiding:**
```
"[Company/creator/tool] chahte hain yeh mat pata chale. [The thing they're not advertising]."
```

**Formula C — Conventional Wisdom Contradiction:**
```
"Sabne bola [conventional advice]. Galat hai. [Correct approach] — personally test kiya hai."
```

**AI_SNIPP Adapted Examples:**

1. "Claude ki koi nahi batata yeh setting. Custom Instructions mein ek field hai jo sab kuch badal deta hai."

2. "Koi nahi bata raha — Anthropic API ka yeh feature free tier mein hai. Seekho iske baad."

3. "Tutorial mein batate hain 'ChatGPT ke saath seekho.' Seedha Claude API pe jaao — 3x faster learning."

4. "Sabne bola AI-generated content detect ho jaata hai. Yeh setting karo — tools dhundh nahi paate."

5. "ProductHunt pe tools dhundh rahe ho? Wrong source. Yeh 3 websites hain jo 6 months pehle tools list karti hain."

6. "Sabne bola portfolio chahiye AI jobs ke liye. GitHub contributions usse zyada important hain — proof hai."

**Hook intensity:** Medium-high — creates insider identity without manufactured drama.
**Best for:** F01, F04, F09
**Avoid for:** F07 (news should be straightforward, not conspiratorial)

### CATEGORY 6 — SYSTEM/RESULT HOOKS ("3-step system...")

**Pattern:** Opens with the promise of a complete, executable system. The viewer's mental model: "I don't have to figure this out — this person has already built the system for me."

**When to use:** F09 (Three Step System), F06 (AI vs Human — when showing the system not just the result), F01 (when the cheat code is part of a workflow).

**Tone:** SYSTEM PROMISE acting direction. Measured confidence. Not excited — confirmed. The creator has built and tested this. The delivery is the calm of someone reporting a known result, not pitching a new idea.

**Formula A — System Named:**
```
"Yeh [name]-step AI system hai jo [specific result]. Save karo pehle, phir dekho."
```

**Formula B — Time Promise:**
```
"[Task] mein [time] lagte the. Is system se [reduced time]. [The 3 steps in one line]."
```

**Formula C — Problem → System:**
```
"[Problem] ka permanent solution hai. [N] steps — aaj implement karo."
```

**AI_SNIPP Adapted Examples:**

1. "Yeh 3-step AI system hai jo tumhara cold email reply rate teen guna kar dega. Save karo pehle, phir dekho."
   *(From EX03 — F09: Three Step System)*

2. "Mere Claude workflow ne resume review time 45 min se 8 min kiya. 3 steps — sab ke liye kaam karta hai."

3. "AI se code review karne ka ek complete system banaya hai. Har bug, har edge case — yeh system dhundh leta hai."

4. "Interview prep ke liye yeh 3-step Claude system hai. 2 hafte mein confidence double ho jaata hai."

5. "Content calendar 4 ghante mein banana tha — ab 25 minute. System batata hun step by step."

6. "LinkedIn optimize karne ka AI system — 3 prompts, 30 minute, aur profile top 10% mein aata hai."

**Hook intensity:** Medium — reliable, trustworthy, not shocking.
**Best for:** F09, F06
**Save karo pehle:** Add this as a second line for any system hook — pre-sells the CTA before viewer even knows the value.

### CATEGORY 7 — VS/COMPARISON HOOKS ("ChatGPT vs Claude...")

**Pattern:** Sets up an explicit comparison. Creates instant curiosity — "which side wins?" Also positions the creator as objective analyst who has done the comparison so the viewer doesn't have to.

**When to use:** F06 (AI vs Human), F04 (Hidden Tool — when comparing tools), F01 (when comparing with-and-without the trick).

**Tone:** Matter-of-fact. Slightly elevated — this is interesting. Not sensationalist.

**Formula A — Tool vs Tool:**
```
"[Tool A] vs [Tool B] — [specific task]. Dono test kiye. [Unexpected winner] jeeta."
```

**Formula B — Manual vs AI:**
```
"Manual [task] — [time]. AI se — [reduced time]. [Fark dekhao]."
```

**Formula C — Before vs After:**
```
"Pehle yeh tha — [bad state]. Ab yeh hai — [good state]. [What changed — AI tool/prompt]."
```

**AI_SNIPP Adapted Examples:**

1. "ChatGPT vs Claude — code debugging pe. Same bug, same codebase. Claude ne 3 minute mein fix kiya. ChatGPT 12 minute."

2. "Manual resume — 2 ghante. AI se — 15 minute. Quality compare karte hain."

3. "Perplexity vs Google — research ke liye. 50 tabs kholne ki zaroorat khatam ho gayi."

4. "Hinglish content — bina AI, 45 min. Claude se, 8 min. Aur quality better hai."

5. "Free Claude vs Claude Pro — actual performance difference. Kya Pro worth it hai Indian users ke liye?"

6. "ChatGPT-4o vs Gemini 2.5 Flash — same prompt, same task. Jo baat koi nahi kar raha woh batata hun."

**Hook intensity:** Medium — creates curiosity without alarm.
**Best for:** F06, F04
**Important rule:** Never declare a winner the creator hasn't actually tested. The Indian tech audience will test claims and call out inaccuracy publicly.

### CATEGORY 8 — CONFESSION/ADMISSION HOOKS

**Pattern:** Creator admits a past mistake or suboptimal behavior. Creates immediate relatability — "even the expert was getting this wrong." Viewer thinks: "If they were doing it wrong, I definitely am."

**When to use:** F01 (Cheat Code — when the trick is a correction to a common bad habit), F09 (System — when the system replaced a worse approach), F08 (Transformation — setting up the before state).

**Tone:** Slightly self-deprecating. Warm humor. Not embarrassed — wiser in retrospect.

**Formula:**
```
"Confession: Main [wrong approach] karta tha. [How long]. [What changed everything]."
```

**AI_SNIPP Adapted Examples:**

1. "Confession: 3 mahine tak ChatGPT pe kaam kiya kyunki Claude 'alag' lag rahi thi. Yeh mistake mat karo."

2. "Honest: Main pehle 10 tools subscribe karta tha. Ab sirf 3 hain. Yeh 3 sab handle karte hain."

3. "Mujhe admit karna padega — mera pehla Claude prompt 200 words ka tha. Isiliye bad output aata tha."

4. "1 saal tak resume manually likhta raha. AI se try kiya toh 40 minute se 12 minute. 1 saal waste kiya."

5. "6 mahine tak wrong Claude model use karta tha. Haiku use karna tha Opus ki jagah. Cost 8x tha."

**Hook intensity:** Medium — builds trust, creates relatability.
**Best for:** F01, F09, F08 (before state setup)
**Avoid for:** F07 (news doesn't need confession framing)

### CATEGORY 9 — QUESTION HOOKS (Rhetorical / Direct)

**Pattern:** Opens with a question the viewer is already asking themselves (rhetorical) or genuinely invites the viewer to answer (engagement). Rhetorical questions create identification; direct questions create interaction.

**When to use:** F10 (Comment React — feature a real viewer question), F09 (System — position the question as what the system answers), F02 (Future Shock — the question creates the threat).

**Formula A — Rhetorical:**
```
"[Task] 20 minute mein karte ho? AI se 4 minute mein hota hai. [How]."
```

**Formula B — Viewer State:**
```
"Kya tumhara [problem is happening]? Yeh kyun ho raha hai — aur fix kaise karein."
```

**Formula C — Comment Feature:**
```
"[Viewer handle] ne poocha: '[exact question]'. Jawaab hai — [one-line answer]. Let me explain."
```

**AI_SNIPP Adapted Examples:**

1. "Resume tailoring mein 2 ghante lagaate ho har application pe? Main dikhata hun how to do it in 12 minutes."

2. "Claude se response generic aata hai? Yeh ek setting hai jo sab kuch badal deti hai."

3. "@rahul_dev123 ne poocha: 'Claude aur ChatGPT mein actual difference kya hai?' Answer — yeh hai."

4. "Ek AI tool sikhne mein kitna time lagata hai? Ek specific tool — 90 minutes. Sab kuch yaad rehne ke liye."

5. "AI se content generate karo toh robotic lagta hai? Yeh 3 words prompt mein add karo."

**Hook intensity:** Low-medium — accessible, conversational.
**Best for:** F10, F09
**Note for F10:** Always feature the actual commenter's username in the hook (if real) — creates authentic community feel.

### CATEGORY 10 — FUTURE SHOCK HOOKS ("2026 mein...")

**Pattern:** Time-anchored urgency. Creates a deadline the viewer can feel. Unlike generic urgency hooks, Future Shock hooks name a specific date or timeframe that makes the threat concrete.

**When to use:** F02 (Future Shock — primary formula), F07 (News Flash — when news has a future implication). Max 1× per week.

**Tone:** URGENCY but restrained. Not panicked. The creator is a calm senior colleague who has read the data and is delivering a measured warning.

**Formula A — Timeline:**
```
"[Timeframe] mein [AI capability/job change]. [Indian specific implication]. Yeh hai action plan."
```

**Formula B — Deadline:**
```
"[Company/platform] [change] [timeframe]. [Who is affected]. [What to do before the deadline]."
```

**Formula C — Comparative Future:**
```
"Jo aaj [skill/tool] seekh raha hai — [future state]. Jo nahi seekh raha — [other future state]."
```

**AI_SNIPP Adapted Examples:**

1. "2026 mein AI coding assistants sabke workflow mein honge. Jo sirf Java jaanta hai, band ho jaayega. Yeh 3 skills seekho."

2. "Google ne announce kiya — AI search 2026 mid mein India mein roll out hoga. SEO professionals ke liye yeh matlab hai..."

3. "Claude API ki free tier 3 mahine mein band hogi. Jo project banana chahte ho — abhi shuru karo."

4. "Jo AI tools seekhta hai 2025 mein, woh 2026 mein team lead hoga. Jo nahi seekhta — freelancer rakha jaayega."

5. "18 mahine mein AI junior developers ko replace kar sakta hai. Jo senior developer banna chahte hain — yeh roadmap hai."

**Hook intensity:** High — only deploy when the claim is 100% verifiable.
**Frequency:** Maximum once per week — strictly enforced.
**Credibility rule:** Every claim must have a traceable source. If the source can't be cited, the hook cannot be used.

## HOOK TESTING PROTOCOL

### 1-Second Test
Read only the first 3 words of the hook. Ask: "Do I want to know what comes next, without any context?"
- Yes → passes
- No → revise. Change the first 3 words.

### Uniqueness Test
Could this exact hook appear on any other creator's account?
- No → passes
- Yes → add something specific to @ai_snipp's voice (Hinglish, Indian context, "Bete...")

### Source Test
Did this hook come from real audience pain language?
- Yes (traceable to real comment or data) → passes
- No (invented) → mark as provisional, validate against real comment language before using

### The Read-Aloud Test
Read the hook out loud in the creator's voice at fast Hinglish pace. Does it sound natural?
- Yes → passes
- No → the sentence structure is probably too English or too formal. Rewrite.
